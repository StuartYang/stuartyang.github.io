redis面试

为什么redis能这么快？

  • 完全基于内存，绝大部份请求是存粹的内存操作，执行效率高。
  • 数据结构简单，对数据操作也简单。
  • 采用单线程，单线程也能处理高并发请求，想多核也可启动多实例。
  • 使用多路I/O复用模型，非阻塞IO

redis的常用数据

  • string ：最基本的数据类型，二进制安全。
  • hash：string元素组成的字典，适合用于存储对象
  • list：列表，按照string元素插入顺序排序。
  • set：string组成的无序不能重复集合。
  • sorted set：通过分数为集合的大小进行排序。分数可以重复，元素不能重复。

从海量key里查询出某一固定前缀的key

要注意细节：摸清数据规模，即问清楚边界

  • keys指令一次性返回所有匹配的key
  • 键的数量过大会使服务卡顿

scan 指令不会使服务器卡住，适合生产环境中

  • 以0作为游标开始一次新的迭代，直刀命令返回游标0完成一次遍历。
  • 不保证每次执行都返回某个数量的元素，支持模糊查询。
  • 一次返回的数量不可控，只能大概率符合count参数。
  • 得到的keys可能是重复的
